# tic-tac-toe

Репозиторий: **https://github.com/akarakuts/tic-tac-toe**

```bash
git clone https://github.com/akarakuts/tic-tac-toe.git
cd tic-tac-toe
```

Нативная игра **«Крестики-нолики»** для **macOS** (линия из *n* подряд на квадратном поле), на **SpriteKit** и **Swift**. Два игрока за одним экраном или один человек против компьютера. Интерфейс поддерживает разный размер поля и условия победы.

Полная версия на английском: [`README.md`](README.md).

## Возможности

- **Размер поля** — от **3×3** до **6×6**, выбор в левой колонке настроек.
- **Условие победы** — длина линии от **3** до **6**; значения больше стороны поля недоступны (должно быть ≤ размера доски). Учитываются строки, столбцы и обе диагонали.
- **Два игрока** — поочерёдные ходы на одном поле (первыми ходят ✕).
- **Против компьютера** — минимакс с альфа-бета отсечением; можно выбрать сторону — крестики (✕) или нолики (○). На поле **3×3** при трёх в ряд ИИ обходит всё дерево игры (идеальная игра). На **больших полях** поиск ограничен по глубине с позиционной эвристикой и немедленными ходами «победа / блок», чтобы игра оставалась отзывчивой.
- **Вёрстка** — элементы управления (размер поля, длина линии, режим, сторона против ИИ, **Новая игра**) слева; **сетка расширяется** и занимает оставшееся место окна. Окно масштабируется вместе с представлением, обрезания за краем экрана нет.
- **Локализация** — строки интерфейса на **английском** и **русском** через `Localizable.xcstrings`; активный язык следует системной локали.
- **Оформление** — анимация ходов, подсветка выигрышной линии, лёгкая обратная связь при ничьей и исходе партии.

## Требования

- macOS (целевая версия и SDK — в проекте Xcode)
- Xcode со Swift 5+ и поддержкой String Catalog

## Сборка и запуск

1. Откройте `tic-tac-toe.xcodeproj` в Xcode.
2. Выберите схему **tic-tac-toe** и **My Mac**.
3. Запуск (**⌘R**).

Сборка из командной строки:

```bash
xcodebuild -scheme "tic-tac-toe" -destination 'platform=macOS' build
```

## Тесты

Юнит-тесты в таргете **tic-tac-toeTests** (Swift Testing). Пример:

```bash
xcodebuild -scheme "tic-tac-toe" -destination 'platform=macOS' -only-testing:tic-tac-toeTests test
```

Правила поля покрыты тестами `GameModel`; поведение ИИ (в том числе не только 3×3) — `TicTacToeAITests`.

## Структура проекта

| Путь | Назначение |
|------|------------|
| `tic-tac-toe/GameModel.swift` | Состояние доски, ходы, поиск выигрышной линии |
| `tic-tac-toe/TicTacToeAI.swift` | ИИ на минимаксе, режим противника, ограничение глубины на больших полях |
| `tic-tac-toe/GameScene.swift` | Сцена SpriteKit: раскладка, сетка, ввод, HUD, эффекты |
| `tic-tac-toe/ViewController.swift` | Хостинг сцены в `SKView`, размеры и режим масштаба |
| `tic-tac-toe/L10n.swift` | Доступ к локализованным строкам |
| `tic-tac-toe/Localizable.xcstrings` | Строки EN и RU |

## Релизы и установка из GitHub

При пуше **тега** GitHub Actions собирает конфигурацию **Release** и прикладывает архив **`tic-tac-toe-<тег>-macOS.zip`** с приложением `tic-tac-toe.app` к [Release](https://github.com/akarakuts/tic-tac-toe/releases). Сборка без подписи разработчика: при первом запуске может потребоваться открыть через контекстное меню («Открыть»).
